## (os) 리눅스 성능분석을 위한 명령어 2
> free, df

<br>

### free
> 시스템의 메모리 정보 출력

```bash
$ free -m 
# -m : 단위를 메가바이트로 출력
```

- free : 어느 누구도 사용하고 있지 않은 메모리 
- available : 애플리케이션에게 실질적으로 할당 가능한 메모리 (= free + buff/cache)
- buff : 블록 디바이스가 가지고 있는 블록 자체에 대한 캐시 
- cache : I/O 성능 향상을 위해 사용하는 페이지 캐시 
- buff/cache 는 I/O 성능 향상을 위해 존재 

#### 페이지 캐시 
> 블록 디바이스 → 페이지 캐시 → 애플리케이션 A 
>                           → 애플리케이션 B 

- 블록 디바이스에 읽은 동일한 내용을 페이지 캐시를 통해 읽을 수 있다 -> I/O 성능 향상
- 한 번 읽었던 파일은 페이지 캐시에 저장함으로써 블록 디바이스가 아닌 메모리에서 파일의 내용을 가져온다 
- 애플리케이션에서 메모리를 필요로 하면 buff/cache 영역을 해제하고 애플리케이션이 사용할 수 있는 영역으로 바꾼다 
- buff/cache 가 높다면 I/O 가 많이 일어나는 환경 -> 더 많은 메모리를 줘서 성능을 향상시킨다

#### swap
> 프로세스 A → 커널 - swap out → swap 영역
> 프로세스 B →      (← swap in -)

- 메모리가 부족한 상황에서 사용되는 가상 메모리 공간으로, 주로 블록디바이스의 일부 영역을 사용
- 프로세스 B 가 커널에 메모리 요청하면 프로세스 A 의 메모리를 swap out 하고 프로세스 B 의 메모리를 할당
- swap in 이 되어도 swap 영역에서 지워지지 않고 이 영역을 swap cached 라고 부른다
- swap 영역이 사용되면 I/O 가 빈번하게 발생하기 때문에 성능 저하 발생
- 컨테이너는 빠르게 다시 띄울 수 있기 때문에 swap 을 써서 성능이 저하될 바에는 swap 영역을 비활성화하는 경우가 많다 (ex> ec2 는 디폴트가 swap X)

### df
> 디스크 여유 공간 및 inode 공간 확인

```bash
$ df -h
```

#### 디렉터리별 사용량 측정
```bash
$ du -sh ./*
# / 디렉 하위 디렉들의 사용량 측정
```

- 2GB 정도 되는 파일을 지웠음에도 디스크 사용량이 그대로인 경우가 있음 → 파일 핸들
- 다른 프로세스에서 참조하고 있는 등의 파일 핸들 확인 명령어

```bash
$ sudo lsof | grep [directory-name]
```

#### inode 사용률
- 파일 또는 디렉에 대한 메타데이터를 저장하는 구조체 = 파일과 디렉터리의 개수

```bash
$ df -i
```
- `fs.file-max` : 동시에 파일을 몇개까지 열 수 있는가를 정의하는 값으로, 커널 파라미터
